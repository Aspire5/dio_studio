# Future Ideas

Ideas for potential features or improvements. These are not committed to. They exist as a parking lot for discussion and evaluation.

Moving an idea to the roadmap requires explicit approval and a decision record.

---

## Ideas

### Request/Response Diffing
Compare two recorded API sessions to identify differences. Useful for regression testing or verifying API contract changes.

### HAR Export
Export recorded traffic in HTTP Archive (HAR) format for use in browser dev tools or external analysis tools.

### CLI Companion Tool
A command-line tool that reads recorded API sessions and provides analysis, search, or replay capabilities outside the app.

### GraphQL Support
Extend mocking and recording to understand GraphQL queries and mutations natively, not just as raw HTTP requests.

### OpenAPI Schema Validation
Validate requests and responses against an OpenAPI specification file. Flag mismatches during development.

### Dio Instance Health Check
A diagnostic feature that inspects a Dio instance and reports its configuration, registered interceptors, and potential issues.

### Traffic Dashboard Widget
A Flutter widget that displays live API traffic for debugging during development. Overlay-style, similar to network inspectors in browser dev tools.

### Performance Metrics
Track response times, payload sizes, and error rates per endpoint. Surface trends and anomalies.
